## Домашнее задание №1 «Hello, OTUS!»

Необходимо написать программу, печатающую в стандартный вывод перевернутую фразу

```
Hello, OTUS!
```

Для переворота строки следует воспользоваться возможностями
[golang.org/x/example/hello/reverse](https://pkg.go.dev/golang.org/x/example/hello/reverse).

Кроме этого необходимо исправить **go.mod** так, чтобы для данного модуля работала
команда `go get`, а полученный **go.sum** закоммитить.

### Критерии оценки

- Пайплайн зелёный - 4 балла
- Используется `reverse` - 4 балла
- Понятность и чистота кода - до 2 баллов

#### Зачёт от 7 баллов

### Подсказки

- `Reverse`

### Как я делал (заметки для себя)

Ниже — не официальная инструкция, а то, как я проходил задание и что для меня в итоге оказалось важным. Может пригодиться, если с Go сталкиваешься впервые.

#### 1. Go

Скачал с [go.dev/dl](https://go.dev/dl/), поставил. Проверка:

```bash
go version
```

Если команда не находится — значит, Go не в PATH или не установлен; без этого дальше смысла нет.

#### 2. `main.go`

Я оставил обычный `package main` и одну `func main()`. Подключил `fmt` для вывода в консоль и пакет `reverse` из [golang.org/x/example/hello/reverse](https://pkg.go.dev/golang.org/x/example/hello/reverse) — по условию через него и нужно переворачивать строку. В `main` у меня что-то вроде `fmt.Println(reverse.String("Hello, OTUS!"))` — в терминал уходит уже перевернутый текст.

#### 3. `go.mod`

В `module` добавил путь своего репозитория. После этого `go get` ведёт себя нормально. Строка `go 1.23` — минимальная версия.

#### 4. Зависимость и `go get`

Из папки `hw01_hello_otus`:

```bash
go get golang.org/x/example/hello/reverse
```

Go сам подтянул модуль, дописал `require` в `go.mod` и обновил `go.sum`. Вариант, который я ещё пробовал: сначала написать `import`, потом `go mod tidy` — тоже работает.

#### 5. `go.sum`

Появился после `go get`. Я его закоммитил вместе с `go.mod`, как в условии: там хеши версий, чтобы у всех сборка была с теми же зависимостями.

#### 6. Запуск

```bash
go run .
```

Точка — «текущая папка». Ожидаемый вывод для фразы из задания: `!SUTO ,olleH`.

#### 7. `go mod tidy`

Иногда гоняю перед коммитом — убирает лишнее из `go.mod` и подтягивает недостающее. Не обязательно каждый раз, но мне так спокойнее.

#### 8. `test.sh`

Это не `go test`, а bash-скрипт: гоняет `go run main.go`, через `sed` подчищает пробелы по краям и сверяет с `!SUTO ,olleH`. Если всё ок — в конце будет `PASS`.

Запускать из каталога `hw01_hello_otus` (чтобы `main.go` был «рядом»).

На Linux/macOS я делал так:

```bash
cd /путь/к/hw01_hello_otus
bash test.sh
```

На Windows из PowerShell/cmd `.sh` сам по себе не запускается — я открываю **Git Bash** (идёт с [Git for Windows](https://git-scm.com/download/win)), перехожу в папку. Путь к диску C у меня выглядит как `/c/...`, например:

```bash
cd "/c/MyDocs/Programming_courses/Golang/yuit-docs-golang/hw01_hello_otus"
bash test.sh
```

У тебя путь будет свой; главное — оказаться в той же папке, где лежит `main.go`.

WSL тоже подойдёт, но только если уже поставлен нормальный дистрибутив; у меня без него `wsl` только ругался, что Linux не установлен.

Нужны Go и `sed` (в Git Bash он уже есть).
